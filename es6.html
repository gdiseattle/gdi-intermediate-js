Variable Naming

Let

// The let statement declares a block scope local variable, optionally initializing it to a value.
							// using var, which hoists and does not use block scope
							let y = 1;

							if (y === 1) {
							  let y = 2;

							  console.log(y);
							  // expected output: 2
							}

							console.log(y);
							// expected output: 1

Try assigning PI to another number and see what the console says

Var
							var x = 1;

							if (x === 1) {
							  var x = 2;

							  console.log(x);
							  // expected output: 2
							}

							console.log(x);
							// expected output: 2

Variable Naming

Constants

Support for constants (also known as "immutable variables"), i.e., variables which cannot be re-assigned new content. Notice: this only makes the variable itself immutable, not its assigned content (for instance, in case the content is an object, this means the object itself can still be altered). 

ES6
							const PI = 3.141593

Try assigning PI to another number and see what the console says

ES5
							//  only in ES5 through the help of object properties
							//  and only in global context and not in a block scope
							Object.defineProperty(typeof global === "object" ? global : window, "PI", {
							    value:        3.141593,
							    enumerable:   true,
							    writable:     false,
							    configurable: false
							})

Function Syntax

// ES6 uses arrow functions (also called fat arrow functions) to shorten syntax.
							// An arrow function does not have it's own this, arguments, super, or new.target and cannot be used as a constructor.

							// ES5 version
							function add(a, b) {
								return a + b;
							};

							// ES6 arrow function
							add = (num1, num2) => {
							 return num1 + num2
							}

							// even shorter es6 syntax
							add = (num1, num2) => (
							  num1 + num2
							)

							// even shorter
							add = (num1, num2) => num1 + num2
